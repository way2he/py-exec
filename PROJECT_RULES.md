# 项目规范总览

## 规范文件索引

本项目采用模块化的规范管理方式，主要规范文件如下：

1. [代码风格规范](docs/rules/code_style.md)
   - Python 代码规范
   - 命名规范
   - 注释规范
   - 导入规范
   - 类型注解

2. [Git 工作流规范](docs/rules/git_workflow.md)
   - 分支管理
   - 提交规范
   - 代码审查

3. [测试规范](docs/rules/testing.md)
   - 单元测试
   - 集成测试
   - 测试覆盖率
   - 测试命名

4. [安全规范](docs/rules/security.md)
   - 代码安全
   - 数据安全
   - 访问控制
   - 安全审计

## 核心原则

1. **代码质量**
   - 保持代码简洁清晰
   - 遵循 DRY 原则
   - 注重代码可维护性
   - 重视代码复用

2. **开发流程**
   - 遵循 Git Flow 工作流
   - 严格执行代码审查
   - 保持文档同步更新
   - 定期进行代码重构

3. **团队协作**
   - 及时沟通
   - 互相学习
   - 共同进步
   - 知识共享

## 规范执行

1. **新成员入职**
   - 阅读所有规范文档
   - 理解项目架构
   - 熟悉开发流程
   - 掌握工具使用

2. **日常开发**
   - 遵循代码规范
   - 执行测试要求
   - 保持文档更新
   - 参与代码审查

3. **规范更新**
   - 定期审查规范
   - 收集团队反馈
   - 及时更新文档
   - 确保规范适用

## 工具支持

1. **代码检查**
   - flake8
   - pylint
   - mypy
   - black

2. **测试工具**
   - pytest
   - coverage
   - tox
   - hypothesis

3. **文档工具**
   - Sphinx
   - MkDocs
   - Read the Docs

## 注意事项

1. 规范不是一成不变的，可以根据项目发展进行调整
2. 遇到规范未覆盖的情况，及时讨论并更新规范
3. 规范执行需要团队成员的共同维护
4. 定期回顾和优化规范内容
